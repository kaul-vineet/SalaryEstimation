CLASSIFICATION PROBLEMS IN MACHINE LEARNING 

This app demonstrates how machine learning models could be hosted on Heroku as REST APIs. Downstream applications could use the APIs to make predictions.

GENERAL INSTRUCTIONS ON HOW TO USE/DEMO PREDICTION API:

  • This app is a HEROKU hosted prediction model for estimating salary from demographic features?

 

  • The app implements Decision Tree supervised machine learning model.

 

  • The model leverages census data to predict individual income. The predicted income is classified as "Above" or "Below" the threshold value of "50K".

 

  • Data is sourced from UCI machine learning repository here.

  • OPTIONAL: If you want to create your own HEROKU app from the repository, follow the steps below:
    • Click GitHub - SalaryEstimation to access repository for the project.
    • Go to Click on the README.md section and click on the "Deploy to Heroku" button (referenced in blue outline below).

 

  • The image below provides a sample HTTP REST request and response. 
    • Call POST method "predictsalary". Reference full URL in black outline. If you have copied the repository into your own app, replace the URL/name of the app (referenced in red underline) with your app name.
    • Create JSON payload with feature name/value pair.

  • The app returns two possible classification values as a response -
    • Income more than 50K
    • Income less than 50K

 

  • Test and demo the model by randomly selecting sample data from UCI machine learning repository here.
